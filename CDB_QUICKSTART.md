# CDB快速使用指南

## 🚀 5分钟快速上手

### 步骤1: 安装TinyCDB

```bash
# Debian/Ubuntu
sudo apt-get install tinycdb

# CentOS/RHEL
sudo yum install tinycdb

# 验证
cdb -h && echo "✓ 安装成功"
```

### 步骤2: 生成CDB索引

```bash
# 运行工具
./sangforfw_log.sh

# 选择菜单
3. 更新索引

# 输出示例：
# 正在更新索引...
# 正在生成CDB索引...
# ✓ CDB索引生成成功
#   文本索引: 5.2M
#   CDB索引: 4.8M
```

### 步骤3: 使用CDB加速查询

```bash
# 运行工具
./sangforfw_log.sh

# 选择菜单
2. 高级查询
1. 单IP查询

# 输入查询条件
请输入IP地址：2.55.81.95
请输入日期（YYYY-MM-DD，留空查询所有日期）：2026-04-27

# 输出示例：
# [CDB加速模式]
# 正在搜索...
# 查询完成，耗时: 0.04秒
# 找到 1,234 条记录
```

## 🎯 使用技巧

### 技巧1: 何时使用CDB？

✅ **推荐场景**
- 查询特定日期的IP记录
- 高频查询（每分钟>10次）
- 多用户并发查询
- 生产环境

❌ **不适用场景**
- 查询所有日期（留空日期）
- 时间范围查询（跨多天）
- 双IP查询、端口查询等其他查询类型

### 技巧2: 查看CDB状态

```bash
# 运行工具后，主菜单会显示：
═══════════════════════════════════════
  防火墙日志查询工具 v2.1
  © 2026 QJKJ Team
═══════════════════════════════════════

  索引: 66,871 条 | 日志: 1 个
  CDB: ✓ 已启用 (快速查询)    ← CDB状态
```

状态说明：
- `✓ 已启用` - CDB已生成，可用
- `○ 可用但未生成` - 已安装TinyCDB但未生成索引
- `✗ 未安装` - 未安装TinyCDB

### 技巧3: 定期更新CDB索引

```bash
# 方式1: 手动更新
./sangforfw_log.sh
选择：3. 更新索引

# 方式2: 定时任务（推荐）
crontab -e

# 每天凌晨2点更新索引
0 2 * * * /path/to/sangforfw_log.sh --rebuild-index

# 每30分钟增量更新
*/30 * * * * /path/to/sangforfw_log.sh --update-index
```

## 🔍 故障排查

### 问题1: 查询时未显示"CDB加速模式"

**原因**：
- 未安装TinyCDB
- 未生成CDB索引
- 未指定日期（留空）

**解决**：
```bash
# 1. 检查TinyCDB
which cdb cdbmake

# 2. 检查CDB索引文件
ls -lh data/index/sangfor_fw_log_index.cdb

# 3. 重新生成索引
./sangforfw_log.sh
选择：3. 更新索引

# 4. 查询时必须指定日期
请输入日期：2026-04-27  ← 不要留空
```

### 问题2: CDB索引生成失败

**检查磁盘空间**：
```bash
df -h
# 确保有足够空间（至少是文本索引的1.2倍）
```

**手动测试CDB**：
```bash
# 创建测试CDB
echo "+3,5:abc->12345" | cdbmake test.cdb test.cdb.tmp

# 查询测试
cdb -q test.cdb abc
# 应输出: 12345

# 清理
rm -f test.cdb test.cdb.tmp
```

### 问题3: 性能未提升

**确认使用了CDB模式**：
- 查询时应显示 `[CDB加速模式]`
- 如果显示 `[传统查询模式]`，说明未使用CDB

**检查CDB文件完整性**：
```bash
# 查看CDB统计信息
cdb -s data/index/sangfor_fw_log_index.cdb

# 输出示例：
# number of records: 66871
# data size: 4892160
# hash table size: 524288
```
