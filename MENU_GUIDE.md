## 功能详解

### 1. 快速查询（基于索引）

**适用场景**：
- 查询特定IP的所有日志
- 查询特定端口的访问记录
- 查询特定协议的流量
- 组合查询（IP+端口、IP+协议等）

**性能特点**：
- 速度极快（毫秒级响应）
- 基于预建索引，无需扫描原始日志
- 支持多字段组合查询

**使用示例**：
```bash
# 查询单个IP
fwlog -t 192.168.1.100

# 查询IP+端口
fwlog -t 192.168.1.100:443

# 查询IP+协议
fwlog -t 192.168.1.100 --protocol TCP

# 查询端口范围
fwlog -t :8000-9000
```

**注意事项**：
- 首次使用需先建立索引（选项3）
- 索引不包含日志全文，仅包含关键字段
- 如需全文搜索，请使用高级查询

---

### 2. 高级查询（全文搜索）

**适用场景**：
- 需要搜索日志全文内容
- 复杂的组合条件查询
- 索引中未包含的字段查询
- 正则表达式匹配

**性能特点**：
- 速度较慢（需扫描原始日志）
- 支持任意字段和内容搜索
- 支持正则表达式

**使用示例**：
```bash
# 全文搜索关键词
高级查询 > 输入关键词: "connection refused"

# 组合条件
高级查询 > 输入IP: 192.168.1.100
         > 输入日期范围: 2026-01-01 to 2026-01-31
         > 输入关键词: "deny"
```

**注意事项**：
- 查询时间与日志文件大小成正比
- 建议先缩小日期范围再查询
- 大量结果会自动分页显示

---

### 3. 重建索引（全量）

**适用场景**：
- 首次使用工具
- 索引损坏或不一致
- 日志目录发生重大变更
- 升级到新版本后

**功能说明**：
- 扫描所有日志文件
- 提取关键字段（IP、端口、协议、日期等）
- 构建多字段索引
- 自动压缩索引文件

**性能特点**：
- 支持并行构建（4-8核加速2-6倍）
- 实时显示进度条
- 自动压缩（节省50-80%空间）

**使用建议**：
- 首次使用必须执行
- 日常维护使用"增量更新"即可
- 大量日志时建议在空闲时段执行

**预计时间**：
- 1GB日志：约1-2分钟（并行模式）
- 10GB日志：约10-20分钟（并行模式）
- 100GB日志：约2-3小时（并行模式）

---

### 4. 增量更新索引

**适用场景**：
- 日常维护（推荐）
- 新增少量日志文件
- 现有日志文件有追加内容
- 定期更新索引

**功能说明**：
- 仅处理变更的文件
- 自动检测新增、修改、删除的文件
- 保留未变更文件的索引
- 速度比全量重建快10-100倍

**性能对比**：
```
场景：50个日志文件，新增2个文件

全量重建：扫描50个文件，耗时5分钟
增量更新：仅扫描2个文件，耗时10秒

速度提升：30倍
```

**使用建议**：
- 每天定时执行（如每晚凌晨）
- 新增日志后立即执行
- 配合cron实现自动化

**自动化示例**：
```bash
# 添加到crontab，每天凌晨2点自动更新
0 2 * * * /path/to/sangforfw_log.sh --incremental-update
```

---

### 5. 快捷键管理

**功能说明**：
- 安装快捷键：将`fwlog`命令注册到系统
- 卸载快捷键：移除快捷键配置
- 查看状态：检查快捷键是否已安装

**安装后效果**：
```bash
# 无需输入完整路径
fwlog                    # 打开主菜单
fwlog -t 192.168.1.100   # 直接查询
fwlog --help             # 查看帮助
```

**支持的Shell**：
- Bash（~/.bashrc）
- Zsh（~/.zshrc）

**注意事项**：
- 安装后需重启终端或执行`source ~/.bashrc`
- 卸载后快捷键立即失效
- 脚本移动位置后需重新安装

---

### 6. 清理导出文件

**功能说明**：
- 清理查询结果导出文件
- 支持按类型分类清理
- 显示清理统计信息

**导出文件分类**：
```
export/
├── by_ip/          # IP查询结果
├── by_port/        # 端口查询结果
├── by_date/        # 日期查询结果
└── by_query/       # 高级查询结果
```

**清理选项**：
1. 清理所有导出文件
2. 仅清理IP查询结果
3. 仅清理端口查询结果
4. 仅清理日期查询结果
5. 仅清理高级查询结果

**使用建议**：
- 定期清理，避免占用过多磁盘空间
- 重要结果请先备份
- 清理前会显示文件数量和大小

**示例输出**：
```
清理导出文件
─────────────────────────────────────────────────────
1. 清理所有导出文件（125个文件，450MB）
2. 仅清理IP查询结果（50个文件，180MB）
3. 仅清理端口查询结果（30个文件，120MB）
4. 仅清理日期查询结果（25个文件，90MB）
5. 仅清理高级查询结果（20个文件，60MB）
0. 返回主菜单

请选择清理选项：
```

## 使用流程建议

### 首次使用
1. 执行选项3（重建索引）
2. 执行选项5（安装快捷键）
3. 使用选项1（快速查询）测试

### 日常使用
1. 使用`fwlog -t <IP>`快速查询
2. 定期执行选项4（增量更新）
3. 定期执行选项6（清理导出文件）

### 故障排查
1. 索引异常：执行选项3（重建索引）
2. 快捷键失效：执行选项5（重新安装）
3. 查询无结果：检查日志目录和索引状态

## 快捷键命令参考

```bash
# 打开主菜单
fwlog

# 快速查询
fwlog -t <IP>                    # 查询IP
fwlog -t <IP>:<PORT>             # 查询IP+端口
fwlog -t :<PORT>                 # 查询端口
fwlog -t <IP> --protocol <PROTO> # 查询IP+协议

# 日期范围查询
fwlog -t <IP> -d 2026-01-01      # 单日查询
fwlog -t <IP> -d 2026-01-01:2026-01-31  # 范围查询

# 索引管理
fwlog --rebuild-index            # 重建索引
fwlog --incremental-update       # 增量更新

# 快捷键管理
fwlog --install-shortcut         # 安装快捷键
fwlog --uninstall-shortcut       # 卸载快捷键

# 帮助信息
fwlog --help                     # 查看帮助
```

## 性能优化建议

### 查询优化
1. 优先使用快速查询（选项1）
2. 缩小日期范围
3. 使用具体的IP而非通配符
4. 定期更新索引

### 索引优化
1. 使用增量更新代替全量重建
2. 启用索引压缩（自动）
3. 定期清理无效索引
4. 并行构建（自动检测）

### 存储优化
1. 定期清理导出文件
2. 启用索引压缩
3. 归档旧日志文件
4. 监控磁盘空间

## 常见问题

### Q1: 索引构建很慢怎么办？
A: 
- 检查是否启用并行模式（自动检测）
- 使用增量更新代替全量重建
- 在系统空闲时段执行
- 考虑升级硬件（CPU、磁盘）

### Q2: 查询结果不准确？
A:
- 检查索引是否最新（查看更新时间）
- 执行增量更新或重建索引
- 使用高级查询进行全文搜索验证

### Q3: 快捷键不生效？
A:
- 检查是否已安装（选项5）
- 重启终端或执行`source ~/.bashrc`
- 检查脚本路径是否正确
- 重新安装快捷键

### Q4: 磁盘空间不足？
A:
- 清理导出文件（选项6）
- 启用索引压缩（自动）
- 归档或删除旧日志
- 检查是否有重复索引文件
