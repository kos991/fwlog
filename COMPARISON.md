## 🔧 技术架构对比

### 旧版本
```
日志文件 → 文本索引(grep) → 全文扫描 → 结果
```
- 索引格式：纯文本（IP|日期|路径|端口|协议）
- 查询方式：grep全文扫描
- 存储方式：行式存储，无压缩
- 性能瓶颈：磁盘I/O

### 新版本
```
日志文件 → CSV临时文件 → DuckDB COPY → 列式存储 → 索引查询 → 结果
```
- 索引格式：DuckDB列式数据库
- 查询方式：SQL索引查询
- 存储方式：列式存储 + 自动压缩
- 性能优势：内存缓存 + 索引加速

## 🎓 迁移指南

### 从旧版本迁移到新版本

1. **备份数据**
```bash
cp -r data/sangfor_fw_log /backup/
```

2. **安装新版本**
```bash
chmod +x setup.sh
sudo ./setup.sh
```

3. **复制日志文件**
```bash
cp /backup/sangfor_fw_log/*.log /data/sangfor_fw_log/
```

4. **建立索引**
```bash
systemctl start fw-log-query
# 或手动运行
./build/fw_log_query -rebuild
```

5. **验证功能**
```bash
# 访问Web界面
curl http://localhost:8080

# 测试查询
curl "http://localhost:8080/api/query?ip=192.168.1.1"
```
